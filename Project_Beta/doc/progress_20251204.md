### 3. モジュールキャッシュ問題の解決

#### 問題

2台同時走行時、Robot1とRobot2が同じ`perception_Startsignal`モジュールを共有してしまい、一方がスタート検出すると他方が検出できなくなる問題が発生。

#### 原因

`detect_start_signal`関数の状態（`ready_to_go`属性）が関数オブジェクトに紐づいており、通常のimportではモジュールがキャッシュされて共有される。

#### 解決策

`importlib.util`を使用して、各ロボット専用のモジュールとして分離読み込み：

```python
import importlib.util
from pathlib import Path

_this_dir = Path(__file__).parent
_startsignal_path = _this_dir / "rule_based_algorithms" / "perception_Startsignal.py"
_startsignal_spec = importlib.util.spec_from_file_location(
    f"perception_Startsignal_{_this_dir.name}",  # ユニークなモジュール名
    _startsignal_path
)
_startsignal_module = importlib.util.module_from_spec(_startsignal_spec)
_startsignal_spec.loader.exec_module(_startsignal_module)
detect_start_signal = _startsignal_module.detect_start_signal
```

## 技術的知見

### Pythonモジュールキャッシュ

- `sys.modules`にキャッシュされたモジュールは、異なるパスからimportしても同一インスタンス
- 関数属性（`func.attr`）を状態管理に使う場合、モジュール共有で問題が発生
- `importlib.util`で明示的にモジュール名を分けることで独立したインスタンスを作成可能

### ROIキャリブレーションツール

