# 開発進捗レポート - 2024年11月30日

## 概要

E2E（End-to-End）AIモード（MODE_NUM=4）を実装し、1周完走を達成。

| ファイル | 説明 |

### 2. Hybrid戦略の実装

**目的**: False Start問題の解決

- ルールベースのスタート信号検出（`perception_Startsignal.py`）
- AI駆動制御（学習済みモデル）
- 戦略を`ai_control_strategy.py`で切り替え可能

### 3. トレーニングパイプライン改善

- StartSequenceデータの自動除外機能
- GPUトレーニング対応（CUDA 12.4）
- データ品質管理の仕組み

## 解決した問題

### Problem 1: False Start

| 項目 | 内容 |
|------|------|
| 症状 | 赤信号中に小さなトルク出力 |
| 原因 | モデルが完全ゼロ出力を学習困難 |
| 解決 | Hybrid戦略（ルールベース+AI） |

### Problem 2: 低トルク出力

| 項目 | 内容 |
|------|------|
| 症状 | Drive出力が0.03〜0.05で車が動かない |
| 原因 | AI失敗データ（drive≈0）がトレーニングに混入 |
| 解決 | 不良データ除外、キーボードデータのみで再トレーニング |

## 技術的知見

### データ品質の重要性

- トレーニングデータの分布がモデル出力に直結
- AI失敗走行データは除外が必須
- `drive_torque`平均 > 0.3 が良質データの目安

### Hybrid戦略の有効性

- 信頼性の高いルールベース処理（スタート検出）
- 柔軟性のあるAI制御（走行）
- 組み合わせることで両者の利点を活用
